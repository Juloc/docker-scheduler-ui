# Skill: UI changes

Use this for UI work.

1. Read `docs/DESIGN.md`.
2. Reuse design tokens/components before adding CSS.
3. Keep work surfaces flat/compact; reserve elevation for dashboard summaries/popovers.
4. Verify System/Dark/Light in all changed surfaces.
5. Check keyboard focus, icon accessible names and mobile layout.
6. Do not add gradients/glass/hero patterns.
7. Remove superseded CSS/components after migration; do not leave parallel design systems.
